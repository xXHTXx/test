Blatt 1: Grundlagen Linux

Aufgabe 1: Start von Diensten

Wie wird ein Dienst (= Task, Programm, Service) Debian?

Um einen Dienst auf Debian zu erstellen, schreibt man ein Skript, erstellt eine Systemd-Dienstdatei in /etc/systemd/system/, lädt Systemd mit sudo systemctl daemon-reload, und startet den Dienst mit sudo systemctl start <dienstname>

Was macht Systemd eigentlich?

Systemd ist ein System- und Dienstmanager für Linux, der:

  • Systemstart und -stopp steuert,
  • Dienste verwaltet und überwacht,
  • Parallelisierung für schnelleren Boot ermöglicht,
  • Abhängigkeiten zwischen Diensten handhabt,
  • Logging von Systemereignissen bereitstellt,
  • Konfigurationsdateien für Dienste verwendet.

Welche Rolle hat das Verzeichnis /etc und etc/systemd?

/etc: Beinhaltet systemweite Konfigurationsdateien.
/etc/systemd: Enthält Systemd-Konfigurationsdateien für die Verwaltung von Diensten und Systemprozessen.

Wofür brauch man sysctl?

sysctl wird verwendet, um Kernel-Parameter zur Laufzeit zu lesen und zu ändern, ohne das System neu zu starten. Es ermöglicht die Anpassung von Kernel-Einstellungen und Systemverhalten, wie z.B. Netzwerk- und Speicherparameter.

Was ist eine chroot-Umgebung?

Eine chroot-Umgebung isoliert einen Prozess, indem sie sein Wurzelverzeichnis auf ein bestimmtes Verzeichnis festlegt, um Sicherheit zu erhöhen und den Zugriff auf das restliche System zu verhindern.

Wozu dient das Verzeichnis /dev/null und /lib/systemd?

/dev/null: Verwirft Eingaben und ignoriert Ausgaben.
/lib/systemd: Beinhaltet Systemd-Bibliotheken und -Konfigurationen.

Aufgabe 2: Environment von Diensten

Was ist der Unterschied von ’user’ und ’group’ und welchen Systemdateien werden hierfür eingesetzt?

  • User (Benutzer): Ein individueller Account, der auf dem System arbeitet und eigene Dateien und Berechtigungen hat. Jeder Benutzer hat eine eindeutige Benutzer-ID (UID).

  • Group (Gruppe): Eine Sammlung von Benutzern, die gemeinsame Berechtigungen für Dateien und Verzeichnisse haben. Jede Gruppe hat eine eindeutige Gruppen-ID (GID).

Was ist eine ’Shell’ und wie kann sie gesetzt werden?

Eine Shell ist ein Programm, das als Benutzeroberfläche zur Interaktion mit dem Betriebssystem dient und Befehle ausführt.

Was macht ulimit und locale?

ulimit: Setzt oder zeigt Ressourcengrenzen für Prozesse des aktuellen Benutzers an (z.B. maximale Anzahl offener Dateien).

locale: Zeigt oder ändert die Ländereinstellungen und Spracheinstellungen für Formatierungen und Zeichencodierung an.

Nutzen Sie env und überlegen Sie, wie Sie in der Shell Environment-Variabeln setzen, lesen, überschreiben und löschen können!

Setzen von Umgebungsvariablen:

Um eine neue Umgebungsvariable zu setzen, verwenden wir den Befehl export:
export MY_VAR="Hello World"

Lesen von Umgebungsvariablen:

Um den Wert einer Umgebungsvariable anzuzeigen, können wir echo oder printenv verwenden:
printenv MY_VAR

Überschreiben von Umgebungsvariablen:

Um eine Umgebungsvariable zu überschreiben, benutzen wir erneut export:
export MY_VAR="New Value"

Löschen von Umgebungsvariablen:

Um eine Umgebungsvariable zu löschen, verwenden wir unset:
unset MY_VAR

Welche Rolle hat hierbei die Shell?

Die Shell verwaltet Umgebungsvariablen, indem sie deren Werte setzt (export), liest (echo, printenv), überschreibt (export) und löscht (unset). Sie sorgt dafür, dass diese Variablen korrekt an Programme und Skripte weitergegeben werden, die innerhalb der Shell ausgeführt werden.

Was sind ’Pfade’ und wie werden sie gesetzt?

Pfade sind Verzeichnispfade, die von der Shell verwendet werden, um Dateien und Programme zu finden. Sie werden durch die Umgebungsvariable PATH definiert.

Was versteht man unter einer ’Shell expansion’?

Shell Expansion bezeichnet die automatische Ersetzung und Umwandlung von Platzhaltern oder Variablen durch die Shell vor der Ausführung eines Befehls.
echo $HOME

Was ist ein shebang?

Ein Shebang (auch als "Hashbang" bezeichnet) ist eine spezielle Zeichenfolge am Anfang eines Skripts, die angibt, welches Programm verwendet werden soll, um das Skript auszuführen. Er besteht aus den Zeichen #! gefolgt vom Pfad zu dem Interpreter-Programm.
#!/bin/bash

Warum ist das Aufsetzen von sudo wichtig?

Um sudo auf einem Debian-System zu setzen, stellen Sie sicher, dass der Benutzer die notwendigen Berechtigungen hat, um Befehle als Superuser auszuführen.
sudo apt update

  • Sicherheitskontrolle: Erlaubt gezielte Befehlsausführung als Superuser, ohne das gesamte System als Root zu betreiben.
  • Verwaltung von Berechtigungen: Präzise Steuerung, welche Befehle mit erhöhten Rechten ausgeführt werden können.
  • Verfolgung von Aktivitäten: Protokolliert alle sudo-Befehle zur Überwachung und Nachverfolgung.

Aufgabe 3: Logging von Diensten

Was versteht man unter Unix-Dateideskriptoren und wofür werden sie genutzt?

Unix-Dateideskriptoren sind numerische Kennungen, die Prozessen zugewiesen werden, um auf Dateien, Pipes oder Sockets zuzugreifen. Sie ermöglichen das Lesen, Schreiben und Verwalten von Datenströmen.

  • Standard-Deskriptoren:
    • 0 für stdin (Eingabe)
    • 1 für stdout (Ausgabe)
    • 2 für stderr (Fehlerausgabe)

Was unterscheidet FD2 von FD1?

FD1 und FD2 sind Unix-Dateideskriptoren:

  • FD1 (Standard-Ausgabe): Für reguläre Programmausgaben.
  • FD2 (Standard-Fehler): Für Fehlermeldungen und Warnungen.

UnDiagnostik mit Wiresharkterschied: FD1 leitet normale Ausgaben weiter, FD2 leitet Fehler weiter.

Welche Zeitstempel besitzt eine Unix-Datei?

Eine Unix-Datei hat drei Zeitstempel:

  • ctime: Letzte Änderung der Datei-Metadaten.
  • mtime: Letzte Änderung des Inhalts.
  • atime: Letzter Zugriff auf die Datei.

Was versteht man unter ’append write’?

Append Write bezeichnet das Anhängen von Daten an das Ende einer Datei, ohne den vorhandenen Inhalt zu überschreiben.

Warum ist Log-Rotating notwendig?

Log-Rotating ist notwendig, um:

  • Speicherplatz zu sparen.
  • Systemleistung zu erhalten.
  • Log-Dateien leichter zu verwalten.

Was passiert bei einem Dienst, der keine Log-Information mehr schreiben kann?

Wenn ein Dienst keine Log-Informationen mehr schreiben kann:

  • Fehlermeldungen gehen verloren.
  • Aktivitäten werden nicht protokolliert.
  • Der Dienst könnte Fehlfunktionen zeigen.

Aufgabe 3: Signale, Pipes, Sockets

Unter welchem Benutzer/Gruppe läuft der Prozess? (Unter Linux kommt die cgroup hinzu)

Ein Prozess läuft unter einem Benutzer und einer Gruppe und ist in einer cgroup für die Ressourcenverwaltung eingeordnet.

Wichtige Environment-Variablen sind:

Variable Erklärung
PATH Definiert die Verzeichnisse, in denen nach ausführbaren Dateien gesucht wird
HOME Gibt das Home-Verzeichnis des Benutzers an
USER Enthält den Benutzernamen
SHELL Gibt die Standard-Shell des Benutzers an
PWD Zeigt das aktuelle Arbeitsverzeichnis an
LANG estimmt die Lokalisierungseinstellungen und Sprache
EDITOR Gibt den Standard-Editor an

Für welche Signale ist der child Prozess empfänglich?

Ein Kindprozess ist in der Regel für die folgenden Signale empfänglich:

Variable Erklärung
SIGINT Interrupt-Signal (z.B. durch Strg+C)
SIGTERM Terminierungs-Signal (sanfter Abschied)
SIGKILL Sofortiges Beenden (kann nicht abgefangen oder ignoriert werden)
SIGQUIT Beenden und Core-Dump erzeugen (z.B. durch Strg+)
SIGSTOP Anhalten des Prozesses (kann nicht abgefangen oder ignoriert werden)
SIGCONT Fortsetzen eines angehaltenen Prozesses

Blatt 3: Linux SystemD

Aufgabe 1: Umfang der systemD Konfiguration

Welche Dienste werden von SystemD verwaltet?

Verwaltung: Systemd verwaltet Systemdienste und -prozesse, einschließlich Netzwerksdiensten, Daemonen und Systemdiensten.

Wie sind die Konfigurationsdateien verzahnt?

Konfigurationsdateien sind in /etc/systemd/system/ und /lib/systemd/system/ gespeichert. Einheitendateien (Unit-Files) definieren Dienste, Sockets, Mount-Punkte usw.

Protokollieren Sie alle notwendigen Prozesse?

Systemd protokolliert alle relevanten Prozesse in journald, zugänglich über journalctl.

Wie sieht ein typischer unit-File aus?

[Unit]
Description=Mein Dienst

[Service]
ExecStart=/usr/bin/mein-dienst
Restart=always

[Install]
WantedBy=multi-user.target

Wie wurde der Apache unit-File aufgesetzt?

Apache wird typischerweise über /etc/systemd/system/apache2.service konfiguriert.

Welche Dienste müssen vor dem Start von Apache aktiv sein und welche im Anschluss?

Dienste vor dem Start von Apache:

network.target: Netzwerk muss bereit sein, um HTTP-Anfragen zu empfangen.
remote-fs.target: Falls Remote-Dateisysteme verwendet werden, müssen diese bereit sein.

Dienste nach dem Start von Apache:

Keine spezifischen Dienste erforderlich: Apache selbst übernimmt die Webanfragen und ist meist unabhängig von anderen Diensten nach dem Start.

Wie wird der Start/Shutdown von Apache unterstützt (z.B. bei Änderung der Konfigurationsdatei)?

Der Start/Shutdown von Apache wird über Systemd unterstützt durch:

Starten:
sudo systemctl start apache2

Stoppen:
sudo systemctl stop apache2

Neustarten bei Konfigurationsänderungen:
sudo systemctl restart apache2

Neuladen der Konfiguration ohne Neustart:
sudo systemctl reload apache2

Diese Befehle verwalten den Dienst und übernehmen das Laden neuer Konfigurationen ohne kompletten Neustart.

Aufgabe 2: Quellenstudium für systemD

  • systemd-system.conf: Globale Systemd-Konfigurationseinstellungen.
  • systemctl: Befehl zur Steuerung und Verwaltung von Systemd-Diensten.
  • systemd-notify: Benachrichtigt Systemd über den Status eines Dienstes.
  • systemd.unit: Definitionen für Systemd-Dienste und -Einheiten.
  • systemd.special: Beschreibung spezieller Systemd-Zustände wie shutdown.target.
  • systemd.exec: Konfigurationsoptionen für die Ausführung von Befehlen in Unit-Dateien.

Aufgabe 4: systemd – apache2.service

Ersetzen Sie dies durch den generischen Aufruf des httpd-Daemons.

Neuer Unit-File für httpd:
Dateipfad: /etc/systemd/system/httpd.service

Beschreiben Sie im unit-File die notwendigen Abhängigkeiten und Voraussetzungen.

[Unit]
Description=Apache HTTP Server
# Stellt sicher, dass der Dienst erst startet, nachdem das Netzwerk bereit ist
After=network.target
# Der Dienst ist auf das Netzwerk angewiesen;
# ohne Netzwerk wird der Dienst nicht gestartet
Requires=network.target

[Service]
# Startet den httpd-Daemon im Vordergrund
ExecStart=/usr/sbin/httpd -DFOREGROUND
# Lädt die Konfiguration neu, ohne den Dienst neu zu starten
ExecReload=/usr/sbin/httpd -k graceful
# Beendet den httpd-Daemon
ExecStop=/usr/sbin/httpd -k stop
# Startet den Dienst automatisch neu, falls er fehlschlägt
Restart=on-failure
# Setzt die Umgebungsvariable für die Locale
Environment=LANG=en_US.UTF-8

[Install]
# Aktiviert den Dienst beim Erreichen des Multi-User-Runlevels
WantedBy=multi-user.target

Stellen Sie die notwendigen Environment-Variablen und Konfigurationsdateien ein.

In der Systemd-Unit-Datei /etc/systemd/system/httpd.service:

[Service]
Environment=LANG=en_US.UTF-8

Konfigurationsdateien:

Hauptkonfigurationsdatei:
Debian/Ubuntu: /etc/apache2/apache2.conf

Zusätzliche Konfigurationsdateien:
Debian/Ubuntu: /etc/apache2/sites-available/

Definieren Sie das Logging des Service (evtl. per Syslog).

Standard-Logs: /var/log/apache2/ (Debian/Ubuntu)
Systemd-Logs: Anzeigen mit: journalctl -u httpd

Überprüfen Sie das korrekte Logging mit den verschiedenen Stufen (welche sind diese?).

Log-Stufen:

Shorts Bedeutung
emerg Notfall
alert Alarm
crit Kritisch
error Fehler
warn Warnung
notice Hinweis
info Information
debug Debugging

Aufgabe 2: Diagnostik mit Wireshark

Diagramm

  • Client → ARP → Switch → Router → Internet
  • Client → DNS → Server IP
  • Client → TCP → HTTPS → Server
  • Server → OCSP (über HTTPS) → Client

Physikalische Schicht:
ARP: Ermittelt die MAC-Adressen.

Netzwerkschicht:
IP: Adressierung und Routing von Paketen.

Transportschicht:
TCP: Gewährleistet eine zuverlässige Übertragung.

Sitzungsschicht:
TLS: Verschlüsselt die Datenübertragung.

Anwendungsschicht:
DNS: Übersetzt Domainnamen in IP-Adressen.
HTTP/HTTPS: Überträgt Webinhalte.
OCSP: Überprüft den Status von Zertifikaten.

Erklärungen:

Protocol Filter Erklärung
Address Resolution Protocol arp ARP wird verwendet, um die MAC-Adresse eines Hosts anhand seiner IP-Adresse zu ermitteln. Es arbeitet auf der Data-Link-Schicht (Layer 2)
Internet Protocol ip IP sorgt für die Adressierung und Routing von Paketen zwischen Hosts. Es arbeitet auf der Netzwerkschicht (Layer 3)
Transmission Conrtol Protocol tcp TCP ist ein verbindungsorientiertes Protokoll auf der Transportschicht (Layer 4), das für die zuverlässige Übertragung von Daten sorgt
Transport Layer Security tls TLS verschlüsselt die Datenübertragung auf der Transportschicht und stellt die Vertraulichkeit und Integrität sicher
Domian Name System dns DNS übersetzt Domainnamen in IP-Adressen und arbeitet auf der Anwendungsschicht (Layer 7)
Hypertext Transfer Protocol http HTTP ist das Protokoll für den Austausch von Webseiteninhalten und arbeitet auf der Anwendungsschicht (Layer 7)
HTTP Secure https HTTPS ist HTTP über TLS/SSL und bietet eine gesicherte Kommunikation auf der Anwendungsschicht (Layer 7)
Online Certificate Status Protocol ocsp OCSP wird verwendet, um den Status von Zertifikaten in Echtzeit zu überprüfen. Es läuft oft über HTTPS
